# CLAUDE.md

이 문서는 개인 CS 데이터 분석 위키에서 Claude가 따르는 작업 규칙과 구조를 정의합니다. 이 CLAUDE.md 하나에 위키 목적, 폴더 구조, 데이터 처리 규칙, 노트 작성 규칙, 로그 규칙, 분석 기준이 모두 들어갑니다.

## 위키 목적
- 통신사 고객서비스(CS) 데이터 분석과 인사이트 축적을 위한 개인 위키입니다.
- 이 위키는 raw 데이터 보존, 질문 정의, 데이터 정리, 분석 실행, 인사이트 기록, 작업 로그 관리를 모두 포함합니다.

## 폴더 구조
- `raw/`: 원본 자료를 그대로 보존하는 폴더입니다.
- `00_inbox/`: 정리되지 않은 자료, 검토 요청, 요약 노트를 임시로 저장하는 폴더입니다.
- `01_questions/`: 분석 질문과 필요 데이터, 우선순위 등을 정리하는 폴더입니다.
- `02_data/`: 정리된 데이터 스키마와 데이터셋 설명 노트를 저장하는 폴더입니다.
- `03_analysis/`: 실제 분석 과정, 코드나 통계 결과, 시각화 등을 기록하는 폴더입니다.
- `04_insights/`: 분석 결과로 얻은 인사이트와 시사점을 정리하는 폴더입니다.
- `05_log/`: 작업 세션별 로그를 날짜 단위로 기록하는 폴더입니다.
- 루트 `INDEX.md`: 위키 허브 노트로, 주요 섹션 링크와 새 노트 목록을 관리합니다.

## 데이터 정보
- `raw/` 폴더에는 CS 데이터 5종이 있습니다:
  - `data_customers.csv` : 고객 마스터, 500명
  - `data_voc.csv` : VOC 데이터, 1,307건
  - `data_consultations.csv` : 상담 로그, 1,320건
  - `data_satisfaction.csv` : 만족도 데이터, 1,320건
  - `data_usage_history.csv` : 이용 이력, 6,000건
- 연결 키:
  - 공통 키 `customer_id`로 대부분의 데이터 연결
  - `data_satisfaction.csv`는 `consult_id`로 `data_consultations.csv`와 연결
- 분석 기간: 2024년 1월~12월

## 원본 자료 처리 규칙
- `raw/` 폴더의 파일은 절대 수정하지 않습니다. 원본은 그대로 보존합니다.
- `raw/`에 새 파일이 추가되면 다음 순서로 처리합니다:
  1. `raw/` 파일 목록과 `02_data/`, `00_inbox/` 노트 목록을 파일명 기준으로 대조합니다.
  2. 이미 대응하는 노트가 있는 파일은 다시 읽지 않습니다(토큰 낭비 방지).
  3. 새 파일 중 표 형태 데이터는 `02_data/`에 스키마 노트를 생성합니다.
  4. 새 파일 중 표가 아닌 자유 형식 자료(메모, 텍스트 등)는 `00_inbox/`에 요약 노트를 만듭니다.
  5. 파일 내용이 불분명해서 분류 판단이 어려우면 추측해서 노트를 만들지 말고 `00_inbox/`에 검토 요청 메모를 남깁니다.
  6. 생성한 노트에는 반드시 `source: [[raw/원본파일명]]`을 남깁니다.
  7. 공통 컬럼(예: `customer_id`)이 있는 데이터끼리는 위키링크로 연결합니다.

## 작업 로그 규칙
- 세션이 끝나면 `05_log/오늘날짜.md`에 이번 세션에서 처리한 `raw` 파일과 생성한 노트 목록을 요약해서 기록합니다.

## 노트 작성 규칙
- 모든 노트 상단에 날짜·분류·관련 데이터 메타데이터를 포함한 YAML 프론트매터를 추가합니다.
- 주제 태그(tags)를 frontmatter에 추가합니다. 예: `이탈분석`, `채널효율`.
- 인사이트 노트는 반드시 `근거 → 해석 → 시사점` 순서로 작성합니다.
- 노트 간 연결은 Obsidian 위키링크 형식 `[[파일명]]`으로 합니다.
- 추측과 사실을 구분해서 씁니다. 예: '~로 보임' vs '~임'.
- 새 노트가 생성될 때마다 `INDEX.md`의 해당 섹션에도 링크를 추가합니다.

## 분석 기준
- 재문의율이 20% 이상이면 개선 필요로 판단합니다.
- 표본 30건 미만 세그먼트는 참고용으로만 사용합니다.

## 예시 실행 흐름
1. `raw/data_customers.csv`가 추가되면 `02_data/data_customers_schema.md`를 생성합니다.
2. `raw/data_voc.csv`가 추가되면 `02_data/data_voc_schema.md`를 생성합니다.
3. `raw/data_satisfaction.csv`가 추가되면 `02_data/data_satisfaction_schema.md`를 생성합니다.
4. 분석 질문은 `01_questions/`에 기록합니다.
5. 분석 결과는 `03_analysis/`에 저장하고, 인사이트는 `04_insights/`에 정리합니다.
6. 새로운 노트는 `INDEX.md`에 링크로 추가합니다.

## Claude 역할
- 이 위키에서 작업할 때마다 이 CLAUDE.md 규칙을 자동으로 따릅니다.
- 위키 목적과 폴더 구조를 존중하며, raw 데이터는 보존하고 새 노트는 규칙에 맞게 작성합니다.
- 데이터 연결, 출처, 프론트매터, 태그, 로그 작성 규칙을 일관되게 유지합니다.
